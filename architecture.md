# ポモドーロタイマーWebアプリ アーキテクチャ案

## 1. ディレクトリ構成（例）

```
1.pomodoro/
├── app.py                # Flaskアプリ本体
├── timer.py              # タイマーロジック（テストしやすい関数群）
├── tests/
│   ├── test_timer.py     # Pythonユニットテスト
│   └── test_api.py       # Flask APIテスト
├── static/
│   ├── css/
│   │   └── style.css     # スタイルシート
│   └── js/
│       ├── timer.js      # タイマーのロジック
│       └── timer.test.js # JSロジックのテスト
├── templates/
│   └── index.html        # メイン画面テンプレート
└── README.md             # 簡単な説明
```

## 2. アーキテクチャ概要

### バックエンド（Flask）
- ルーティング：
  - `/`：メイン画面（index.html）を返す
  - 必要に応じてAPIエンドポイント（例：/api/start, /api/stop, /api/status など）を用意
- 状態管理：
  - 基本はクライアント側で管理。サーバー側でセッションやDB保存が必要な場合はAPIで対応
- テンプレートエンジン：
  - Jinja2（Flask標準）でHTMLを動的生成
- ロジック分離：
  - タイマーや業務ロジックは`timer.py`等のモジュールに実装し、テスト容易化
- 依存性注入：
  - DBや外部サービス利用時はDIを意識し、テスト時にモックを差し替えやすくする

### フロントエンド（HTML/CSS/JavaScript）
- UI：
  - index.htmlでモックに沿ったUIを作成
  - style.cssでデザインを調整
- タイマー制御：
  - timer.jsでポモドーロタイマーのロジック（開始・停止・リセット・残り時間表示・進捗バー等）
  - 必要に応じてサーバーと非同期通信（fetch/AJAX）
- ロジック分離：
  - タイマーのロジックはUI操作と分離し、テスト可能な関数として実装
- テストフレームワーク利用：
  - JestやMocha等でロジック部分をテスト

## 3. ユニットテスト容易化の工夫
- ロジックと入出力の分離（FlaskルートやJSイベントハンドラとロジックを分ける）
- API設計（API単位でテスト可能に）
- テスト用ディレクトリの用意（`tests/`や`timer.test.js`など）
- Mock/Stub活用（外部依存部分のテスト容易化）
- CI/CD導入（GitHub Actions等で自動テスト実行）

## 4. 拡張性
- ユーザーごとの履歴保存や認証が必要な場合はDB導入
- API設計を意識しておくと、将来的なSPA化やモバイル対応も容易

## 5. 実装方針
- 最初は「シングルページ＋クライアント側タイマー」でシンプルに
- 必要に応じてAPI追加・状態管理拡張
